# 5.2.2 DOKUMENTASI HALAMAN & ROUTE SISTEM TK TERATAI

## 5.2.2.1 Halaman Login

Halaman login merupakan pintu masuk utama sistem manajemen TK Teratai yang memastikan setiap pengguna terauthentikasi dengan benar sebelum mengakses aplikasi. Pada halaman ini, pengguna (admin, guru, atau orang tua) memasukkan kredensial mereka berupa email dan password, kemudian sistem akan memvalidasi data tersebut dengan database. Jika login berhasil, user akan diarahkan ke dashboard sesuai dengan rolenya (admin, guru, atau orang tua), dan sistem akan mencatat waktu login untuk audit trail. Fitur keamanan seperti password hashing menggunakan bcrypt diimplementasikan untuk melindungi kredensial pengguna.

**Route:**

-   GET / → Menampilkan form login (login)
-   POST /login → Memproses login dan generate session (login.store)

---

## 5.2.2.2 Halaman Lupa Password

Halaman lupa password memfasilitasi pengguna yang kehilangan akses ke akun mereka untuk mereset password dengan aman melalui email verification. Pengguna memasukkan alamat email terdaftar, sistem akan mengirimkan link reset password ke email tersebut dengan token unik yang berlaku terbatas waktu. Setelah klik link, pengguna diarahkan ke form reset password untuk memasukkan password baru. Sistem menggunakan Laravel Mail untuk pengiriman email dan token encryption untuk keamanan proses reset password.

**Route:**

-   GET /forgot-password → Form lupa password (forgot.password)
-   POST /forgot-password → Kirim link reset ke email (forgot.password.store)
-   GET /reset-password/{token} → Form reset password dengan token (password.reset)
-   POST /reset-password → Proses reset password (password.store)

---

## 5.2.2.3 Halaman Dashboard Admin

Dashboard admin adalah pusat kontrol administrator yang menampilkan ringkasan statistik menyeluruh mengenai data sekolah TK Teratai. Dashboard ini menampilkan widget-widget penting seperti total siswa, total guru, total orang tua, total jadwal, dan pengumuman terbaru. Admin dapat melihat overview lengkap sistem dan akses cepat ke berbagai fitur manajemen data seperti siswa, guru, orang tua, jadwal, mata pelajaran, dan pengumuman. Setiap widget dapat diklik untuk drill-down ke halaman detail terkait.

**Route:**

-   GET /admin/dashboard → Halaman dashboard utama admin (admin.dashboard)

---

## 5.2.2.4 Halaman Manajemen Data Siswa (Admin)

Halaman manajemen data siswa memungkinkan admin untuk melakukan CRUD lengkap terhadap data siswa TK Teratai. Admin dapat melihat daftar semua siswa dengan informasi ID siswa, nama, jenis kelamin, tanggal lahir, kelas, dan orang tua yang terhubung. Fitur pencarian dan filter memudahkan admin menemukan siswa tertentu berdasarkan nama, kelas, atau ID. Admin dapat menambah siswa baru dengan mengisi form lengkap termasuk data pribadi, tempat tanggal lahir, alamat, dan menghubungkan dengan akun orang tua. Setiap siswa memiliki ID unik dengan format SIS-XXXXXX yang auto-generated untuk keperluan identifikasi.

**Route:**

-   GET /admin/siswa → Daftar semua siswa (admin.siswa.index)
-   GET /admin/siswa/create → Form tambah siswa baru (admin.siswa.create)
-   POST /admin/siswa → Simpan siswa baru (admin.siswa.store)
-   GET /admin/siswa/{id} → Lihat detail siswa (admin.siswa.show)
-   GET /admin/siswa/{id}/edit → Form edit siswa (admin.siswa.edit)
-   PUT /admin/siswa/{id} → Update data siswa (admin.siswa.update)
-   DELETE /admin/siswa/{id} → Hapus siswa (admin.siswa.destroy)

---

Admin dapat menambah orang tua baru, mengedit informasi kontak, atau menghapus data orang tua yang sudah tidak aktif. Sistem mencegah penghapusan orang tua yang masih memiliki anak aktif terdaftar di sekolah.

Setiap jadwal memiliki ID unik dengan format JDW-XXXXXX dan sistem mencegah bentrok jadwal untuk guru yang sama di waktu bersamaan.

**Route:**

-   GET /admin/jadwal-siswa → Daftar jadwal dengan penugasan siswa (admin.jadwal-siswa.index)

## 5.2.2.15 Halaman Input Nilai (Guru)

Sistem mendukung berbagai jenis penilaian seperti nilai harian, UTS, UAS, dengan skala 0-100. Setiap input nilai tercatat dengan timestamp dan dapat diedit jika terjadi kesalahan input.

**Route:**

-   GET /guru/input-nilai → Daftar nilai yang sudah diinput (guru.input-nilai.index)

Orang tua dapat melihat daftar laporan perkembangan yang telah dikirim oleh guru dengan informasi periode, mata pelajaran, nilai, dan catatan perkembangan. Halaman detail perkembangan menampilkan breakdown nilai per mata pelajaran, trend perkembangan dalam grafik, dan catatan detail dari guru tentang kemajuan dan area yang perlu ditingkatkan. Fitur filter memudahkan orang tua melihat perkembangan berdasarkan periode atau mata pelajaran tertentu.

**Route:**

-   GET /orangtua/perkembangan → Daftar laporan perkembangan (orangtua.perkembangan.index)
-   GET /orangtua/perkembangan/{id} → Detail laporan perkembangan (orangtua.perkembangan.show)

---

## 5.2.2.23 Halaman Perilaku Anak (Orang Tua)

Halaman perilaku anak menampilkan semua catatan perilaku yang telah dibuat oleh guru terhadap anak mereka. Orang tua dapat melihat list catatan perilaku dengan informasi tanggal, jenis perilaku (positif/negatif), guru yang mencatat, dan deskripsi singkat. Halaman detail perilaku menampilkan observasi lengkap dari guru tentang perilaku, moral, dan etika anak selama proses pembelajaran. Fitur ini membantu orang tua memahami perkembangan karakter anak di sekolah dan dapat melakukan follow-up di rumah untuk mendukung pembentukan karakter positif.

**Route:**

-   GET /orangtua/perilaku → Daftar catatan perilaku anak (orangtua.perilaku.index)
-   GET /orangtua/perilaku/{id} → Detail catatan perilaku (orangtua.perilaku.show)

---

## 5.2.2.24 Halaman Kehadiran Anak (Orang Tua)

Halaman kehadiran anak menampilkan riwayat absensi lengkap anak di setiap pertemuan pembelajaran. Orang tua dapat melihat kalender kehadiran dengan status hadir, sakit, izin, atau alpha dalam format tabel atau kalender visual. Terdapat statistik kehadiran seperti persentase kehadiran, total hari hadir, sakit, izin, dan alpha untuk periode tertentu (bulanan/semesteran). Fitur filter memudahkan orang tua melihat kehadiran berdasarkan mata pelajaran atau periode waktu tertentu untuk monitoring kedisiplinan anak.

**Route:**

-   GET /orangtua/kehadiran → Riwayat kehadiran anak (orangtua.kehadiran.index)
-   GET /orangtua/kehadiran/{id} → Detail kehadiran per periode (orangtua.kehadiran.show)

---

## 5.2.2.25 Halaman Jadwal Belajar Anak (Orang Tua)

Halaman jadwal belajar menampilkan jadwal pembelajaran anak untuk seminggu dengan breakdown per hari, mata pelajaran, guru pengampu, dan jam belajar. Orang tua dapat melihat jadwal dalam format tabel mingguan untuk memudahkan planning dan support belajar anak di rumah. Setiap jadwal menampilkan informasi mata pelajaran, nama guru, kelas, waktu mulai dan selesai. Orang tua juga dapat export jadwal dalam format PDF untuk di-print dan ditempel di rumah sebagai reminder untuk anak.

**Route:**

-   GET /orangtua/jadwal → Jadwal belajar anak (orangtua.jadwal.index)
-   GET /orangtua/jadwal/{id} → Detail jadwal per hari (orangtua.jadwal.show)

---

## 5.2.2.26 Halaman Pengumuman (Orang Tua)

Halaman pengumuman menampilkan semua pengumuman yang dipublikasikan oleh admin untuk dibaca oleh orang tua. Orang tua dapat melihat list pengumuman dengan informasi judul, tanggal publikasi, dan preview isi singkat. Orang tua dapat klik pengumuman untuk membaca isi lengkap dan detail tambahan. Pengumuman biasanya berisi informasi penting tentang kegiatan sekolah, event mendatang, perubahan kebijakan, atau instruksi khusus untuk orang tua. Halaman ini read-only dan tidak memiliki fitur edit/delete untuk orang tua.

**Route:**

-   GET /orangtua/pengumuman → Daftar pengumuman untuk orang tua (orangtua.pengumuman.index)
-   GET /orangtua/pengumuman/{id} → Detail pengumuman (orangtua.pengumuman.show)

---

## 5.2.2.27 Halaman Laporan Lengkap (Orang Tua)

Halaman laporan lengkap menampilkan semua laporan perkembangan anak yang telah dikirim oleh guru kepada orang tua. Orang tua dapat melihat daftar laporan dengan informasi periode, guru yang membuat, dan tanggal laporan. Halaman detail laporan menampilkan informasi komprehensif seperti ringkasan nilai per mata pelajaran, riwayat kehadiran, catatan perilaku, dan catatan perkembangan dari guru. Fitur komunikasi via komentar memungkinkan orang tua bertanya atau memberikan feedback kepada guru tentang perkembangan anak, memfasilitasi kolaborasi efektif antara sekolah dan keluarga dalam mendidik anak.

**Route:**

-   GET /orangtua/laporan-lengkap → Daftar laporan lengkap anak (orangtua.laporan-lengkap.index)
-   GET /orangtua/laporan-lengkap/{id} → Detail laporan lengkap (orangtua.laporan-lengkap.show)
-   POST /orangtua/laporan-lengkap/{id}/komentar → Tambah komentar pada laporan (orangtua.laporan-lengkap.komentar.store)

---

## 5.2.2.28 Halaman Profile Edit (All Users)

Halaman profile edit memungkinkan semua user yang sudah login (admin, guru, atau orang tua) untuk melihat dan mengedit informasi profil pribadi mereka seperti nama, email, nomor telepon, dan foto profil. User dapat mengupdate data profil dan sistem akan menyimpan perubahan ke database. Halaman ini dirancang sederhana dengan fokus pada data yang relevan untuk masing-masing role tanpa fitur-fitur yang tidak perlu. Setiap perubahan profil tercatat dalam log aktivitas untuk audit trail.

**Route:**

-   GET /profile/edit → Form edit profil (profile.edit)
-   POST /profile/edit → Update profil user (profile.update)

---

## 5.2.2.29 Halaman Ganti Password (All Users)

Halaman ganti password memungkinkan semua user untuk mengubah password akun mereka demi keamanan. User memasukkan password lama untuk verifikasi, kemudian memasukkan password baru dan konfirmasi password baru. Sistem memvalidasi bahwa password lama benar, password baru memenuhi kriteria keamanan (minimal 8 karakter), dan konfirmasi password sesuai. Setelah berhasil update, sistem akan mengenkripsi password baru dengan bcrypt hashing dan menyimpan ke database. User akan diminta login ulang setelah ganti password untuk keamanan.

**Route:**

-   GET /profile/password → Form ganti password (profile.password)
-   POST /profile/password → Update password user (profile.updatePassword)

---

## 5.2.2.30 Halaman Logout

Proses logout dilakukan ketika user ingin keluar dari sistem TK Teratai. Sistem akan menghapus session, invalidate token, dan redirect user kembali ke halaman login. Logout dapat dilakukan melalui tombol logout di navbar atau menu user dropdown. Sistem mencatat waktu logout dalam log aktivitas untuk audit trail dan security monitoring.

**Route:**

-   POST /logout → Proses logout user (logout)

## Struktur Role & Permission

**Admin** → Full access ke semua halaman dan fitur admin:

-   Dashboard admin dengan statistik sistem
-   CRUD Siswa, Orang Tua, Guru
-   CRUD Jadwal, Mata Pelajaran
-   Kelola Jadwal Siswa per Kelas
-   CRUD Pengumuman

**Guru** → Akses ke fitur pembelajaran dan evaluasi:

-   Dashboard guru dengan jadwal hari ini
-   Lihat Kelas yang Diajar
-   View Data Siswa (read-only)
-   Lihat Jadwal Mengajar
-   Input & Kelola Nilai Siswa
-   CRUD Catatan Perilaku
-   Input & Kelola Absensi
-   Lihat Pengumuman
-   Buat & Kirim Laporan Lengkap ke Orang Tua
-   Komunikasi via Komentar dengan Orang Tua

-   Dashboard orang tua dengan ringkasan anak
-   Lihat Profil Anak
-   Lihat Laporan Perkembangan
-   Lihat Catatan Perilaku Anak
-   Lihat Riwayat Kehadiran
-   Lihat Jadwal Belajar Anak
-   Lihat Pengumuman Sekolah
-   Lihat Laporan Lengkap dari Guru
-   Komunikasi via Komentar dengan Guru
