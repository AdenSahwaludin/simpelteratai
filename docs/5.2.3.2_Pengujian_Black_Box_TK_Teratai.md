# 5.2.3.2 Skenario Pengujian Black Box Sistem TK Teratai

Pengujian black box dilakukan pada fitur-fitur utama sistem manajemen TK Teratai tanpa mempertimbangkan logika kode internal. Pengujian fokus pada input, proses, dan output untuk memastikan setiap fitur berfungsi sesuai yang diharapkan oleh pengguna.

---

## 1. FITUR LOGIN

Pengujian login memverifikasi kemampuan sistem untuk mengautentikasi pengguna dengan berbagai skenario input valid dan tidak valid, termasuk validasi kredensial, penanganan error, dan keamanan akses berdasarkan peran pengguna yang berbeda-beda.

| 2   | Login dengan email dan password guru yang benar      | Email: guru@tkt.com, Password: guru123               | Masuk ke dashboard guru dengan berhasil                | ✓ Sesuai |
| 3   | Login dengan email dan password orang tua yang benar | Email: orangtua@tkt.com, Password: orangtua123       | Masuk ke dashboard orang tua dengan berhasil           | ✓ Sesuai |
| 4   | Login dengan password yang salah                     | Email: admin@tkt.com, Password: salahpassword        | Sistem menampilkan pesan error login gagal             | ✓ Sesuai |
| 5   | Login dengan email yang tidak terdaftar              | Email: tidakterdaftar@tkt.com, Password: password123 | Sistem menampilkan pesan error email tidak ditemukan   | ✓ Sesuai |
| 6   | Login dengan email kosong                            | Email: (kosong), Password: password123               | Sistem menampilkan validasi error email wajib diisi    | ✓ Sesuai |
| 7   | Login dengan password kosong                         | Email: admin@tkt.com, Password: (kosong)             | Sistem menampilkan validasi error password wajib diisi | ✓ Sesuai |
| 8   | Login dengan format email tidak valid                | Email: admintkt, Password: password123               | Sistem menampilkan validasi error format email salah   | ✓ Sesuai |
| 9   | Akses dashboard tanpa login                          | URL: /admin/dashboard (tanpa login)                  | Sistem redirect ke halaman login                       | ✓ Sesuai |
| 10  | Admin mengakses route guru                           | Admin klik URL /guru/dashboard                       | Sistem menolak akses dan redirect ke dashboard admin   | ✓ Sesuai |
